Selective Search for Object Recognition

by Omar Rahman

View My GitHub Profile

Summary

I am going to implement a parallel selective search algorithm for object recognition in images.

Background

One major requirement of an object recognition system is a high quality set of object hypotheses that describe locations for potential objects to be recognized. This paper suggests a bottom up, hierarchical approach to the problem by taking segments of an image and greedily combining neighboring segments using various diversification and similarity strategies. Another paper proposes a method using "edge boxes", in which the number of contours wholly contained in an edge box indicates the likelihood that it contains an object.

Challenges

The first challenge is to determine which method to employ, a decision which will likely be based largely on the amount of parallelism I can exploit from it. In hierarchical approach, there is potential for parallelism here since we can spatially decompose the problem by considering pairs of segments in parallel; however, this will likely not produce the exact same results as the sequential greedy algorithm. Thus, one challenge will be to determine whether this relaxation of the algorithm actually matters.

Resources

Since image regions provide richer features than individual pixels, Uijlings et al. perform image segmentation using a fast method proposed in this paper. Luckily, this paper provides a C++ implementation of their algorithm, which I hope to use as a starting point for my implementation of selective search.

Goals and Deliverables

I hope to complete a working implementation of selective search, which will, given an image, output an image with object proposals bound in boxes. I will first implement the sequential version as proposed in the paper and then implement a parallel version. I'll need some way to evaluate the parallel implementation, since its result will likely differ from the sequential version. Also, if time permits, I will attempt to speed up the image segmentation preprocessing step.

Platform Choice

I have not yet decided on what platform I will run on, maybe CUDA. I will have to think more carefully about the parallelism involved before committing to a platform.
