Parallel CKY Algorithm for Natural Language Parsing

by Omar Rahman

Summary

I will implement a parallel version of the CKY algorithm used for parsing natural language.

Background

Parsing is a fundamental component of many applications in natural language processing, including machine translation, semantic analysis, and information extraction. Since natural language is so vast and ambiguous, attempts at constructing a formal grammar result in hundreds of thousands of nonterminal rules and tens of thousands of terminal rules. The CKY algorithm is a dynamic programming algorithm that parses in a worst case complexity of O(|G|n^3), where |G| denotes the size of the grammar and n denotes the length of the sentence. The size of the grammar dominates the computation, and so there is an opportunity for parallelism across either the grammar rules or symbols. For user-facing systems, real-time parsing is very desirable. The sequential algorithm, however, takes too long. Parallelizing the algorithm could help in achieving real-time parsing.

Challenges

Since CKY algorithm is a dynamic programming algorithm, there are many dependencies that would cause synchronization issues. Also, considering the layout of the data as the upper triangular portion of a square grid, making efficient use of the cache may prove to be difficult without modification of the data structures. The management and movement of data will also be challenging to do efficiently.

Resources

I will be reimplementing some of the methods described in this paper and try to improve upon their results with my own ideas. I will extract a grammar and lexicon from the Berkeley parser.

Goals and Deliverables

I plan write from scratch a parallel parser that will at least match but preferably outperform the results from the aforementioned paper, which achieved a 26 fold speedup from a sequential C-implementation.
